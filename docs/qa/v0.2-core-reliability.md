# v0.2 Core Reliability QA

This checklist exists for the remaining manual QA item in `v0.2 Alpha: Core Reliability`: verifying terminal copy/paste and resize behavior across macOS, Linux, and Windows.

## Platforms

- Windows 11 x64
- macOS Apple Silicon
- macOS Intel
- Linux X11 or XWayland

## Scenarios

Run each scenario for:

- local session
- SSH session
- Telnet session when available
- Serial session when hardware is available

## Copy/Paste

1. Select terminal text with mouse drag.
2. Copy with the platform shortcut:
   - Windows/Linux: `Ctrl+C` only after text selection
   - macOS: `Cmd+C` only after text selection
3. Paste into a local editor and confirm the exact text matches.
4. Paste plain text from a local editor back into the terminal.
5. Confirm multi-line paste arrives in the remote shell without truncation.
6. Repeat after reconnecting or restarting a stopped tab.

## Resize

1. Open a live terminal tab.
2. Resize the main window wider and narrower several times.
3. Resize the window taller and shorter several times.
4. Resize the sidebar and confirm the terminal reflows correctly.
5. Run `stty size` or the platform equivalent in the shell and confirm rows/cols update.
6. Confirm full-screen and restored window states both preserve usable terminal sizing.

## Regression Checks

1. Open terminal search with `Ctrl+F` or `Cmd+F`.
2. Confirm search still works after resizing.
3. Confirm clear/reset actions do not break subsequent paste input.
4. Confirm stopped-session shortcuts still work after resizing.

## Record

Capture for each platform:

- pass/fail for copy
- pass/fail for paste
- pass/fail for resize
- transport used
- notes for any platform-specific glitch
